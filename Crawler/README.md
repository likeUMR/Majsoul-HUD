# 雀魂最小命令行监听器

这个目录已经被精简成只保留一条最短链路：

- 监听雀魂网页客户端的 WebSocket 流量
- 解析对局事件
- 在命令行输出基础信息

保留的输出重点：

- 登录成功提示
- 新局开始
- 初始手牌
- 出牌情况
- 所有玩家的摸切信息
- 吃碰杠等可操作提示
- 副露、杠、拔北
- 和牌、流局、对局结束

不再保留的功能：

- AI 打牌建议
- 自动更新协议文件
- JSONL 落盘
- 参考工程和其它附加功能

## 运行方式

程序入口仍然是 `run.bat`。

1. 双击 `run.bat`
2. 先选 `1` 安装依赖
3. 再选 `2` 启动监听器
4. 浏览器配置代理后打开雀魂网页版

启动后会生成 `crawler_debug.log`，用于记录解析异常和详细调试信息。
同时会自动启动一个左上角 HUD 透明悬浮层，显示和命令行相同的文本日志。

`run.bat` 内置了清华源安装命令，会自动用下面这个源安装依赖：

```text
https://pypi.tuna.tsinghua.edu.cn/simple
```

## 环境要求

- Windows
- Python 3.10 及以上
- 可正常访问雀魂网页版
- `tkinter` 可用

## HUD 说明

当前版本新增了一个 Windows HUD：

- 始终置顶
- 位于左上角
- 只有文字可见，背景透明
- 鼠标点击穿透，不拦截任何操作
- 与命令行同步显示同样的日志

如果你已经装过旧版本依赖，这次需要重新执行一次 `run.bat` 的选项 `1`，安装新增的 `pillow`。

## 代理和证书配置

这个项目不是 OCR，也不是截图识别，而是 `mitmproxy` 抓取并解析雀魂网络协议。

因此你这台电脑第一次使用时，必须完成下面两个配置：

### 1. 浏览器走本地代理

代理地址：

```text
127.0.0.1:8080
```

可以二选一：

1. 用代理插件给浏览器单独设置代理
2. 给 Chrome 单独建一个快捷方式，在目标后面追加：

```text
 --proxy-server=http://127.0.0.1:8080
```

### 2. 安装 `mitmproxy` 证书

1. 先启动 `run.bat`，选择 `2` 让监听器跑起来
2. 用已经走代理的浏览器访问 [http://mitm.it](http://mitm.it)
3. 下载 Windows 证书
4. 双击安装证书
5. 选择“本地计算机”
6. 选择“将所有的证书都放入下列存储”
7. 存储位置选择“受信任的根证书颁发机构”
8. 安装完成后重启浏览器

如果不装这个证书，浏览器虽然可能能连到代理，但 `HTTPS/WSS` 内容无法被正确解密，雀魂对局事件也就抓不到。

## 当前目录保留内容

- `run.bat`: 一键入口

## 常见问题

### 打开雀魂后没有任何输出

优先检查这几项：

1. `run.bat` 是否已经通过选项 `2` 启动
2. 浏览器是否真的走了 `127.0.0.1:8080`
3. 证书是否安装到了“受信任的根证书颁发机构”
4. 是否打开的是网页版雀魂 `https://game.maj-soul.com/`

### 不想装证书行不行

这个精简版仍然是抓网络包，不装证书基本就无法稳定解析雀魂的 `HTTPS/WSS` 流量。如果你后面想彻底避开证书，那就不是这个项目的简化问题了，而是要改成另一条“截图/OCR”技术路线。
